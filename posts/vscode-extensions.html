VS Code 생산성을 높이는 필수 익스텐션

📅 2025년 12월 8일 ⏱ 4분 💬 10

개발 효율을 극대화할 수 있는 VS Code 익스텐션과 설정 팁을 소개합니다. 실제 사용 경험을 바탕으로 정리했습니다.

1. ESLint - 코드 품질 관리

JavaScript와 TypeScript 코드의 품질을 자동으로 검사하고 문제를 표시해줍니다.

// .eslintrc.json 설정 예시
{
  "extends": ["eslint:recommended", "plugin:@typescript-eslint/recommended"],
  "rules": {
    "no-console": "warn",
    "no-unused-vars": "error",
    "@typescript-eslint/explicit-function-return-type": "off"
  }
}

2. Prettier - 코드 포맷터

일관된 코드 스타일을 자동으로 유지해줍니다. 저장 시 자동 포맷 설정을 권장합니다.

// settings.json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}

3. GitLens - Git 기능 강화

코드의 작성자, 변경 이력을 한눈에 볼 수 있어 협업 시 매우 유용합니다.

  • 각 줄의 blame 정보 표시
  • 파일 히스토리 확인
  • 커밋 비교 및 검색
  • 리포지토리 시각화

4. Live Server - 실시간 미리보기

HTML/CSS/JavaScript 변경사항을 실시간으로 브라우저에 반영합니다.

// settings.json
{
  "liveServer.settings.donotShowInfoMsg": true,
  "liveServer.settings.port": 5500,
  "liveServer.settings.CustomBrowser": "chrome"
}

5. Auto Rename Tag - 태그 자동 수정

HTML/JSX에서 여는 태그를 수정하면 닫는 태그도 자동으로 수정됩니다.

<!-- div를 section으로 변경하면 닫는 태그도 자동 변경 -->
<section>
  <h1>제목</h1>
</section>

6. Path Intellisense - 경로 자동완성

파일 경로를 입력할 때 자동 완성을 제공합니다.

import Component from './components/Component'; // 자동완성
<img src="./images/logo.png" /> // 자동완성

7. Better Comments - 주석 강조

주석을 카테고리별로 색상 구분하여 가독성을 높입니다.

// ! 중요한 주석 (빨간색)
// ? 질문이나 의문 (파란색)
// TODO: 할 일 (주황색)
// * 강조 주석 (초록색)

8. Error Lens - 에러 인라인 표시

에러와 경고를 코드 라인에 직접 표시하여 빠르게 문제를 파악할 수 있습니다.

9. Bracket Pair Colorizer 2 - 괄호 색상 구분

중첩된 괄호를 색상으로 구분하여 코드 블록을 쉽게 식별할 수 있습니다.

// VS Code 1.60 이상에서는 기본 기능으로 제공
{
  "editor.bracketPairColorization.enabled": true,
  "editor.guides.bracketPairs": "active"
}

10. Thunder Client - API 테스팅

VS Code 내에서 바로 REST API를 테스트할 수 있습니다. Postman의 경량 대안입니다.

추가 생산성 설정

// settings.json - 유용한 설정들
{
  // 자동 저장
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  
  // 미니맵 표시
  "editor.minimap.enabled": true,
  
  // 탭 크기
  "editor.tabSize": 2,
  
  // 후행 공백 제거
  "files.trimTrailingWhitespace": true,
  
  // 파일 끝 개행 추가
  "files.insertFinalNewline": true,
  
  // 폰트
  "editor.fontFamily": "Fira Code, Monaco, 'Courier New', monospace",
  "editor.fontLigatures": true,
  "editor.fontSize": 14,
  
  // Emmet
  "emmet.includeLanguages": {
    "javascript": "javascriptreact"
  }
}

유용한 단축키

// macOS 기준
Cmd + P          // 파일 빠른 열기
Cmd + Shift + P  // 명령 팔레트
Cmd + D          // 다음 일치 항목 선택 (다중 커서)
Cmd + /          // 주석 토글
Cmd + Shift + F  // 전체 검색
Cmd + B          // 사이드바 토글
Option + Up/Down // 줄 이동
Option + Shift + Up/Down // 줄 복사

결론

올바른 익스텐션과 설정을 사용하면 개발 생산성이 크게 향상됩니다. 하지만 너무 많은 익스텐션은 오히려 VS Code를 느리게 만들 수 있으니, 정말 필요한 것만 설치하는 것을 권장합니다.
